## 2. 파이썬 기초 문법 빠른 참고

> 이 프로젝트에서 자주 나오는 파이썬 문법들을 먼저 설명합니다.

### 📌 타입 힌트 (Type Hints)

```python
# 일반 파이썬 (타입 없음)
def add(a, b):
    return a + b

# 타입 힌트 있음 (이 프로젝트 스타일)
def add(a: int, b: int) -> int:
    return a + b
```

**설명:** `a: int`는 "a는 정수여야 해"라고 힌트를 주는 것입니다.  
`-> int`는 "이 함수는 정수를 반환해"라는 의미입니다.  
코드 자체에는 영향 없지만, 실수를 줄이고 코드를 읽기 쉽게 해줍니다.

```python
# 자주 나오는 타입들
from typing import Optional, list, dict, Any

name: str = "삼성전자"              # 문자열
count: int = 100                   # 정수
price: float = 75000.5             # 소수점 있는 숫자
is_valid: bool = True              # 참/거짓
prices: list[float] = [100, 200]   # 소수점 리스트
info: dict[str, float] = {"종가": 75000.0}  # 딕셔너리
maybe_name: Optional[str] = None  # 값이 있을 수도 없을 수도 있음
anything: Any = "뭐든지"            # 아무 타입이나
```

### 📌 클래스와 객체지향

```python
# 클래스 = 설계도
class Dog:
    def __init__(self, name: str):  # 생성자 (만들 때 실행됨)
        self.name = name            # self. 로 이 객체의 속성 저장
    
    def bark(self) -> str:
        return f"{self.name}이(가) 짖습니다!"

# 객체 = 설계도로 만든 실제 물건
my_dog = Dog("강아지")
print(my_dog.bark())  # "강아지이(가) 짖습니다!"
```

### 📌 추상 클래스 (ABC - Abstract Base Class)

```python
import abc

class 동물(abc.ABC):           # ABC를 상속하면 추상 클래스
    @abc.abstractmethod        # 이 메서드는 반드시 구현해야 함
    def 소리내기(self) -> str:
        pass                   # 구현 없음 - 하위 클래스가 해야 함

class 강아지(동물):
    def 소리내기(self) -> str:  # 반드시 구현!
        return "멍멍"

class 고양이(동물):
    def 소리내기(self) -> str:
        return "야옹"
```

**설명:** 추상 클래스는 "이 기능은 반드시 있어야 해!"라고 강제하는 틀입니다.  
이 프로젝트에서는 모든 전략이 `generate_signal()` 함수를 반드시 구현하도록 강제합니다.

### 📌 데이터클래스 (dataclass)

```python
from dataclasses import dataclass

@dataclass
class 캔들:
    시가: float
    고가: float
    저가: float
    종가: float
    거래량: float

# 이렇게 쓰면 __init__ 자동 생성됨
candle = 캔들(시가=100, 고가=110, 저가=95, 종가=105, 거래량=1000)
print(candle.종가)  # 105
```

**설명:** `@dataclass`는 데이터를 담는 클래스를 쉽게 만드는 도구입니다.  
`__init__`, `__repr__`, `__eq__` 등을 자동으로 만들어줍니다.

### 📌 비동기 프로그래밍 (async/await)

```python
import asyncio

# 동기 방식 (순서대로, 하나씩)
def 데이터_받기():
    # 1초 기다림 → 완료 → 다음으로
    import time
    time.sleep(1)
    return "데이터"

# 비동기 방식 (기다리는 동안 다른 일 가능)
async def 데이터_받기_비동기():
    await asyncio.sleep(1)  # 기다리는 동안 다른 코루틴 실행 가능
    return "데이터"

# 여러 데이터 동시에 받기 (비동기의 장점!)
async def 여러_데이터():
    aapl, samsung, btc = await asyncio.gather(
        데이터_받기_비동기(),  # 세 개 동시 실행!
        데이터_받기_비동기(),
        데이터_받기_비동기(),
    )
    # 3초가 아닌 ~1초만 걸림!
```

**설명:** `async def`로 만든 함수는 비동기 함수(코루틴)입니다.  
`await`는 "이걸 기다리는 동안 다른 일 해도 돼"라는 신호입니다.  
여러 데이터를 동시에 받을 수 있어 훨씬 빠릅니다.

### 📌 열거형 (Enum)

```python
from enum import Enum

class 신호(str, Enum):
    매수 = "buy"
    매도 = "sell"
    중립 = "neutral"

# 사용
my_signal = 신호.매수
if my_signal == 신호.매수:
    print("매수 신호!")
```

**설명:** Enum은 정해진 값들만 쓸 수 있는 타입입니다.  
오타나 잘못된 값 입력을 방지합니다.

### 📌 프로퍼티 (Property)

```python
class 원:
    def __init__(self, 반지름: float):
        self._반지름 = 반지름  # _ 는 "내부용"을 표시
    
    @property
    def 넓이(self) -> float:    # 함수인데 변수처럼 사용 가능
        return 3.14 * self._반지름 ** 2

circle = 원(5)
print(circle.넓이)  # circle.넓이() 가 아님! 괄호 없음
```

### 📌 데코레이터 (Decorator)

```python
# 데코레이터 = 함수를 포장하는 함수
def 시간측정(함수):
    def 래퍼(*args, **kwargs):
        import time
        시작 = time.time()
        결과 = 함수(*args, **kwargs)
        print(f"실행 시간: {time.time() - 시작:.2f}초")
        return 결과
    return 래퍼

@시간측정          # ← 이게 데코레이터 사용법
def 느린함수():
    import time
    time.sleep(1)

느린함수()  # 자동으로 시간이 측정됨!
```

## 3. 핵심 설정 파일들

### 📄 `app/config/settings.py` — 모든 설정의 중심

**목적:** 데이터베이스 접속 정보, API 키, 거래 한도 등 모든 설정값을 한 곳에서 관리합니다.

#### 핵심 개념: pydantic-settings

```python
from pydantic_settings import BaseSettings, SettingsConfigDict
from pydantic import Field

class APIKeySettings(BaseSettings):
    # env_file=".env" 는 .env 파일에서 값을 자동으로 읽겠다는 설정
    model_config = SettingsConfigDict(
        env_file=".env",
        env_file_encoding="utf-8",
    )
    
    # Field() 로 기본값, 별명, 설명을 설정
    alpha_vantage_key: Optional[str] = Field(
        default=None,                          # 기본값: 없음
        alias="ALPHA_VANTAGE_KEY",             # 환경변수 이름
        description="무료 API 키",              # 설명
    )
```

**설명:**  
- `BaseSettings`를 상속하면 `.env` 파일에서 설정을 자동으로 읽어옵니다.
- `.env` 파일에 `ALPHA_VANTAGE_KEY=abc123` 이라고 적으면 자동으로 `alpha_vantage_key = "abc123"` 이 됩니다.

#### 설정 계층 구조

```
Settings (최상위)
├── DatabaseSettings    ← PostgreSQL, ClickHouse, DuckDB 접속 정보
├── RedisSettings       ← Redis 캐시 설정
├── APIKeySettings      ← 외부 API 키들
├── MLSettings          ← AI 학습 파라미터
├── BacktestSettings    ← 백테스팅 기본값 (수수료, 초기자본 등)
├── RiskSettings        ← 리스크 한도 (최대손실 등)
└── ServerSettings      ← API 서버 설정 (포트, JWT 등)
```

#### `@lru_cache` — 한 번만 초기화하기

```python
from functools import lru_cache

@lru_cache(maxsize=1)   # 결과를 캐싱 (저장)
def get_settings() -> Settings:
    return Settings()   # 처음 호출 때만 만들고, 이후는 저장된 것 반환
```

**설명:** 매번 설정을 다시 읽지 않고 처음 한 번만 읽어서 재사용합니다.  
`maxsize=1`은 하나의 결과만 저장한다는 의미입니다.

### 📄 `app/config/logging_config.py` — 로그 시스템

**목적:** 시스템이 무엇을 하고 있는지 기록(로그)합니다. 문제 발생 시 원인 파악에 필수입니다.

```python
import structlog

# structlog = 구조화된 로그 (JSON 형식으로 저장 가능)
logger = structlog.get_logger(__name__)

# 일반 로그
logger.info("data_fetched", symbol="AAPL", bars=597)
# 출력: {"event": "data_fetched", "symbol": "AAPL", "bars": 597, "timestamp": "..."}

# 에러 로그
logger.error("fetch_failed", symbol="AAPL", error="Network timeout")
```

#### 성능 측정 도구

```python
class PerformanceLogger:
    """코드 실행 시간을 자동으로 기록"""
    
    def __enter__(self):        # with 블록 시작할 때
        self._start = time.time()
        return self
    
    def __exit__(self, *args):  # with 블록 끝날 때
        elapsed = time.time() - self._start
        logger.info(self.name, elapsed_ms=elapsed * 1000)

# 사용법
```

```python
    frozen=True: 한번 만들면 수정 불가 (불변 객체)
    slots=True:  메모리 효율화 (내부 최적화)

    volume: float        # 거래량

            raise ValueError(f"고가({self.high}) < 저가({self.low}) — 잘못된 데이터!")
```

```python
def _validate_data(self, df: pd.DataFrame) -> pd.DataFrame:
    """받은 데이터가 올바른지 확인"""
    
    # 1. OHLC 논리 검증 (고가 ≥ 저가 등)
    invalid_mask = (
        (df["high"] < df["low"]) |        # 고가 < 저가는 불가능
        (df["open"] <= 0) |               # 음수 가격 불가
        (df["close"] <= 0) |
        (df["volume"] < 0)                # 음수 거래량 불가
    )
    df = df[~invalid_mask]  # ~ 는 NOT 연산 (조건 반전)
    
    # 2. 이상치 제거 (5-시그마 법칙)
    returns = df["close"].pct_change()   # 수익률 계산
    mean = returns.mean()
    std = returns.std()
    # 평균에서 5배 표준편차 이상 벗어난 값은 이상치
    outlier_mask = (returns - mean).abs() > 5 * std
    df = df[~outlier_mask]
    
    return df
```

```python
def detect_source(symbol: Symbol) -> DataSource:

    심볼 형태만 보고 어디서 데이터를 가져올지 자동 판단.

    if re.match(r"^\d{6}$", s):

    if "USDT" in s or "/USDT" in s:
        return DataSource.BINANCE
    
    crypto_symbols = {"BTC", "ETH", "BNB", "SOL", "ADA", ...}
    if s in crypto_symbols:

    return DataSource.YAHOO  # 기본값
```

#### 전략 1: 이동평균 크로스 전략

```python
class MovingAverageCrossStrategy(BaseStrategy):
    """
    Golden Cross: 단기 MA가 장기 MA를 위로 돌파 → 매수
    Dead Cross:   단기 MA가 장기 MA를 아래로 돌파 → 매도
    
    예시) 20일 이동평균이 50일 이동평균을 위로 넘으면 → 매수 신호
    """
    
    def __init__(
        self,
        fast_period: int = 20,    # 단기 이동평균 기간
        long_period: int = 50,    # 장기 이동평균 기간
        ma_type: str = "EMA",     # EMA: 최근 데이터 가중, SMA: 단순 평균
    ):
        self.fast_period = fast_period
        self.long_period = long_period
    
    def _calculate_ema(self, prices: np.ndarray, period: int) -> np.ndarray:
        """
        EMA (지수이동평균) 계산.
        최근 데이터일수록 더 많은 가중치를 줌.
        
        multiplier = 2 / (period + 1)
        EMA = 현재가 × multiplier + 이전 EMA × (1 - multiplier)
        """
        multiplier = 2.0 / (period + 1)
        ema = np.empty_like(prices)
        ema[period - 1] = prices[:period].mean()  # 초기값: 단순 평균
        
        for i in range(period, len(prices)):
            ema[i] = prices[i] * multiplier + ema[i-1] * (1 - multiplier)
        
        return ema
    
    def generate_signal(self, data: OHLCVFrame, ...) -> list[Signal]:
        closes = data.close_prices
        fast_ma = self._calculate_ema(closes, self.fast_period)
        slow_ma = self._calculate_ema(closes, self.long_period)
        
        # 어제와 오늘을 비교해서 크로스 감지
        prev_diff = fast_ma[-2] - slow_ma[-2]  # 어제 차이
        curr_diff = fast_ma[-1] - slow_ma[-1]  # 오늘 차이
        
        if prev_diff <= 0 and curr_diff > 0:   # 음수→양수: 위로 돌파
            return [Signal(direction=Direction.LONG, ...)]
        
        if prev_diff >= 0 and curr_diff < 0:   # 양수→음수: 아래로 돌파
            return [Signal(direction=Direction.SHORT, ...)]
        
        return []  # 신호 없음
```

```python
        1. 일별 변화량 계산
        2. 상승일 평균(AU), 하락일 평균(AD) 계산
        3. RSI = 100 - 100/(1 + AU/AD)
        """
        deltas = np.diff(closes)  # 오늘 - 어제
        
        gains = np.where(deltas > 0, deltas, 0)   # 상승한 날만
        losses = np.where(deltas < 0, -deltas, 0)  # 하락한 날만
        
        # Wilder의 평활 이동평균
        avg_gain = gains[:period].mean()
        avg_loss = losses[:period].mean()
        
        for i in range(period, len(gains)):
            avg_gain = (avg_gain * (period - 1) + gains[i]) / period
            avg_loss = (avg_loss * (period - 1) + losses[i]) / period
        
        if avg_loss == 0:
            return 100.0
        
        rs = avg_gain / avg_loss
        return 100.0 - (100.0 / (1.0 + rs))
```

```python
    MACD (Moving Average Convergence Divergence):
    - MACD선 = 12일 EMA - 26일 EMA
    - 시그널선 = MACD선의 9일 EMA
    - MACD선이 시그널선을 위로 돌파 → 매수
    - MACD선이 시그널선을 아래로 돌파 → 매도

        ema26 = self._calculate_ema(closes, 26)
        macd_line = ema12 - ema26              # MACD 라인
        signal_line = self._calculate_ema(macd_line, 9)  # 시그널 라인
        histogram = macd_line - signal_line    # 히스토그램

        if histogram[-2] < 0 and histogram[-1] > 0:
            return [Signal(direction=Direction.LONG, ...)]
        
        if histogram[-2] > 0 and histogram[-1] < 0:
            return [Signal(direction=Direction.SHORT, ...)]
```

```python
class MomentumStrategy(BaseStrategy):
    """
    모멘텀 (Momentum):
    - 최근 12개월간 가격 상승률이 높은 주식이 계속 오를 가능성이 높다는 이론
    - skip-month factor: 최근 1달을 제외 (단기 반전 현상 제거)
    
    계산: 12개월 전 가격 대비 수익률 - 최근 1개월 수익률
    """
    
    def _calculate_momentum(self, closes, lookback=252, skip=21):
        """
        lookback: 12개월 (약 252 거래일)
        skip: 최근 1개월 (약 21 거래일) 제외
        """
        if len(closes) < lookback + skip:
            return 0.0
        
        price_now = closes[-(skip + 1)]       # skip개월 전 가격
        price_past = closes[-(lookback + skip)]  # lookback+skip개월 전 가격
        
        return (price_now - price_past) / price_past  # 수익률
```

```python
class EnsembleStrategy(BaseStrategy):
    """
    여러 전략의 신호를 합쳐서 하나의 신호로 만드는 전략.
    민주주의 투표처럼 다수결로 결정.
    
    예시: MA전략 → 매수, RSI전략 → 매수, MACD전략 → 매도
          매수 2표 vs 매도 1표 → 최종: 매수 (비율 66%)
    """
    
    def __init__(
        self,
        strategies: list[BaseStrategy],  # 전략 목록
        weights: list[float] = None,     # 가중치 (None이면 동일 가중)
        min_agreement: float = 0.5,      # 최소 동의 비율 (50% 이상이어야 신호)
    ):
        self.strategies = strategies
        self.weights = weights or [1.0] * len(strategies)
    
    def generate_signal(self, data, ...):
        all_signals = []
        
        for strategy, weight in zip(self.strategies, self.weights):
            signals = strategy.generate_signal(data)
            for s in signals:
                all_signals.append((s, weight))
        
        if not all_signals:
            return []
        
        # 가중 투표 집계
        long_votes = sum(w for s, w in all_signals if s.is_long)
        short_votes = sum(w for s, w in all_signals if s.is_short)
        total_votes = sum(w for _, w in all_signals)
        
        long_ratio = long_votes / total_votes
        short_ratio = short_votes / total_votes
        
        # min_agreement 이상이면 신호 생성
        if long_ratio >= self.min_agreement:
            return [Signal(direction=Direction.LONG, strength=long_ratio, ...)]
        elif short_ratio >= self.min_agreement:
            return [Signal(direction=Direction.SHORT, strength=short_ratio, ...)]
        
        return []  # 합의 미달
```

**목적:** "만약 이 전략을 과거 5년간 썼다면 얼마나 벌었을까?"를 시뮬레이션합니다.

#### Numba JIT 컴파일

```python
from numba import njit  # JIT = Just-In-Time 컴파일러

@njit(cache=True, fastmath=True)
def _calc_equity_curve(
    signals: np.ndarray,        # 신호 배열 (1=매수, -1=매도, 0=중립)
    close_prices: np.ndarray,   # 종가 배열
    initial_capital: float,     # 초기 자본 (예: 1억원)
    commission_bps: float,      # 수수료 (bps = 0.01%) 
    slippage_bps: float,        # 슬리피지
) -> tuple:
    """
    @njit 는 이 파이썬 함수를 기계어로 컴파일합니다.
    일반 파이썬보다 100~1000배 빠릅니다.
    
    cache=True: 한번 컴파일하면 저장해서 재사용
    fastmath=True: 수학 연산 최적화 (정밀도 약간 낮아도 OK)
    """
    n = len(signals)
    equity = np.empty(n)        # 잔고 추적
    equity[0] = initial_capital
    
    position = 0.0              # 현재 포지션 (0=없음, 1=매수, -1=매도)
    entry_price = 0.0           # 진입 가격
    total_commission = 0.0      # 총 수수료
    
    for i in range(1, n):
        price = close_prices[i]
        signal = signals[i]
        
        if signal != position:  # 포지션 변경 필요
            if position != 0:
                # 기존 포지션 청산
                pnl = (price - entry_price) * position
                commission = abs(equity[i-1]) * commission_bps / 10000
                total_commission += commission
                equity[i] = equity[i-1] + pnl - commission
            else:
                equity[i] = equity[i-1]
            
            if signal != 0:
                # 새 포지션 진입
                slippage = price * slippage_bps / 10000
                entry_price = price + slippage * signal
            
            position = signal
        else:
            # 포지션 유지: 미실현 손익 반영
            if position != 0:
                unrealized = (price - entry_price) * position
                equity[i] = equity[0] + unrealized  # 대략적 추정
            else:
                equity[i] = equity[i-1]
    
    returns = np.diff(equity) / equity[:-1]  # 일별 수익률
    return equity, returns, total_commission
```

**설명:**  
- Numba는 파이썬 코드를 C언어 수준의 속도로 컴파일합니다.
- 일반 for 루프가 100만 번 돌아도 순식간에 완료됩니다.
- `@njit`이 붙은 함수 안에서는 pandas를 못 쓰고 numpy만 사용 가능합니다.

#### 성과 지표 계산

```python
@njit(cache=True, fastmath=True)
def _calc_max_drawdown(equity: np.ndarray) -> float:
    """
    최대 낙폭 (MDD, Maximum Drawdown) 계산.
    고점에서 저점까지 최대 얼마나 떨어졌는지.
    
    예시: 1억 → 1.5억 → 8천만
          MDD = (8천만 - 1.5억) / 1.5억 = -46.7%
    """
    peak = equity[0]
    max_dd = 0.0
    
    for i in range(1, len(equity)):
        if equity[i] > peak:
            peak = equity[i]          # 새 고점 갱신
        drawdown = (equity[i] - peak) / peak
        if drawdown < max_dd:
            max_dd = drawdown         # 더 큰 낙폭이면 갱신
    
    return max_dd  # 음수값 (예: -0.467 = -46.7%)


def calculate_sharpe_ratio(returns: np.ndarray, risk_free_rate: float = 0.02) -> float:
    """
    샤프 비율: 위험 대비 수익률 얼마나 효율적인가?
    
    공식: (연간 수익률 - 무위험 수익률) / 연간 변동성
    
    - 1.0 이상: 양호
    - 2.0 이상: 우수
    - 3.0 이상: 매우 우수 (헤지펀드 수준)
    """
    if len(returns) == 0 or returns.std() == 0:
        return 0.0
    
    annual_return = returns.mean() * 252      # 일별 → 연간 (252 거래일)
    annual_vol = returns.std() * np.sqrt(252) # 일별 변동성 → 연간
    
    return (annual_return - risk_free_rate) / annual_vol
```

```python
class QuantFeatureEngineer:
    """
    40가지 이상의 기술적 특성을 생성하는 클래스.
    
    특성 카테고리:
    - 수익률 관련 (7개): 1일, 5일, 10일, 20일, 60일, 120일, 252일 수익률
    - 기술적 지표 (15개): RSI, ATR, 볼린저밴드, MACD 등
    - 변동성 관련 (5개): 실현 변동성, 변동성의 변동성 등
    - 거래량 관련 (4개): 거래량 비율, OBV 등
    - 모멘텀 (4개): 1개월, 3개월, 6개월, 12개월 모멘텀
    """
    
    def fit(self, train_data: OHLCVFrame) -> "QuantFeatureEngineer":
        """
        훈련 데이터로 정규화 기준값을 계산.
        중요: 미래 데이터를 절대 보면 안 됨! (데이터 누설 방지)
        """
        features = self._create_features(train_data)
        
        # 각 특성의 평균과 표준편차 저장
        self._fit_stats: dict[str, tuple[float, float]] = {}
        for col in features.columns:
            self._fit_stats[col] = (features[col].mean(), features[col].std())
        
        self._is_fitted = True
        return self
    
    def transform(self, data: OHLCVFrame) -> pd.DataFrame:
        """훈련 데이터의 기준값으로 정규화"""
        if not self._is_fitted:
            raise RuntimeError("fit()을 먼저 호출하세요!")
        
        features = self._create_features(data)
        
        # Z-score 정규화: (값 - 평균) / 표준편차
        for col in features.columns:
            mean, std = self._fit_stats[col]
            if std > 0:
                features[col] = (features[col] - mean) / std
        
        return features
```

```python
@njit(cache=True, fastmath=True)
def _rolling_zscore(arr: np.ndarray, window: int) -> np.ndarray:
    """
    롤링 Z-스코어: 현재 값이 최근 window기간 내에서 몇 표준편차인지
    
    예: 오늘 종가가 최근 20일 내에서 +1.5 표준편차
    → 상대적으로 높은 가격이구나!
    """
    result = np.full(len(arr), np.nan)
    
    for i in range(window - 1, len(arr)):
        window_data = arr[i - window + 1: i + 1]  # 슬라이딩 윈도우
        mean = window_data.mean()
        std = window_data.std()
        if std > 0:
            result[i] = (arr[i] - mean) / std
    
    return result


@njit(cache=True, fastmath=True)
def _atr_fast(high, low, close, period=14):
    """
    ATR (Average True Range, 평균 진폭):
    최근 period일간 캔들의 평균 크기.
    변동성 측정에 사용.
    
    True Range = max(고가-저가, |고가-어제종가|, |저가-어제종가|)
    ATR = True Range의 Wilder EMA
    """
    n = len(high)
    tr = np.empty(n)
    
    tr[0] = high[0] - low[0]
    for i in range(1, n):
        # 세 가지 중 최대값
        hl = high[i] - low[i]
        hc = abs(high[i] - close[i-1])
        lc = abs(low[i] - close[i-1])
        tr[i] = max(hl, hc, lc)
    
    # Wilder의 평활 이동평균
    atr = np.empty(n)
    atr[period-1] = tr[:period].mean()
    multiplier = 1.0 / period
    
    for i in range(period, n):
        atr[i] = tr[i] * multiplier + atr[i-1] * (1 - multiplier)
    
    return atr
```

```python
                model.fit(X_train, y_train)
                oof_predictions[val_idx, i] = model.predict(X_val)
        
        # 메타 모델은 OOF 예측값으로 훈련
        self.meta_model.fit(oof_predictions, y)
        return self
```

```python
    """
    Optuna: AI 모델의 설정값(하이퍼파라미터)을 자동으로 최적화.
```

### 📄 `app/risk/risk_manager.py` — 리스크 관리 시스템

**목적:** 주문 전에 여러 안전 기준을 확인해서 과도한 위험을 방지합니다.

#### 체인 오브 체크 (Chain of Checks) 패턴

```python
class RiskManager:
    """
    모든 주문이 통과해야 하는 안전 관문들.
    하나라도 통과 못하면 주문 거부.
    """
    
    def check_order(
        self,
        order: Order,
        portfolio: Portfolio,
        portfolio_value: float,
    ) -> tuple[bool, str]:
        """
        Returns: (통과 여부, 거부 이유)
        """
        
        checks = [
            # 1. 단일 종목 최대 10% 제한
            self._check_position_size(order, portfolio, portfolio_value),
            
            # 2. MDD 15% 초과 시 매수 금지
            self._check_drawdown_limit(portfolio),
            
            # 3. 일일 손실 2% 초과 시 거래 중단
            self._check_daily_loss_limit(portfolio),
            
            # 4. 최대 레버리지 1.5배 제한
            self._check_leverage(portfolio, portfolio_value),
            
            # 5. 특정 종목 집중 방지 (HHI 지수)
            self._check_concentration(order, portfolio),
        ]
        
        # 모든 체크 통과해야 주문 승인
        for passed, reason in checks:
            if not passed:
                return False, reason
        
        return True, "주문 승인"
    
    def _check_position_size(self, order, portfolio, portfolio_value):
        """단일 종목이 포트폴리오의 10% 이상이 되지 않도록"""
        order_value = order.quantity * order.price
        position_ratio = order_value / portfolio_value
        
        if position_ratio > self.max_position_size:  # 기본 0.10
            return False, f"포지션 크기 초과: {position_ratio:.1%} > {self.max_position_size:.1%}"
        return True, ""
```

#### VaR (Value at Risk) — 위험 가치

```python
def calculate_var(
    self,
    returns: np.ndarray,
    confidence: float = 0.95,
    method: str = "historical",
) -> float:
    """
    VaR: "95% 확률로 이 금액 이상 잃지 않는다"는 추정값.
    
    예시: VaR = -2% → 95% 확률로 하루 손실이 2% 이내
    """
    
    if method == "historical":
        # 과거 수익률 분포에서 하위 5% 분위수
        return float(np.percentile(returns, (1 - confidence) * 100))
    
    elif method == "parametric":
        # 정규분포 가정 (빠르지만 꼬리 위험 과소평가)
        from scipy import stats
        mean = returns.mean()
        std = returns.std()
        z_score = stats.norm.ppf(1 - confidence)  # 95% → z = 1.645
        return mean + z_score * std
    
    elif method == "cornish_fisher":
        # Cornish-Fisher 확장: 정규분포 + 왜도/첨도 보정
        # 실제 금융 데이터의 두꺼운 꼬리(fat tail) 반영
        from scipy import stats
        mean = returns.mean()
        std = returns.std()
        skew = stats.skew(returns)    # 왜도 (분포의 비대칭성)
        kurt = stats.kurtosis(returns)  # 첨도 (꼬리의 두꺼움)
        
        z = stats.norm.ppf(1 - confidence)
        # Cornish-Fisher 보정 공식
        z_cf = (z +
                (z**2 - 1) * skew / 6 +
                (z**3 - 3*z) * kurt / 24 -
                (2*z**3 - 5*z) * skew**2 / 36)
        
        return mean + z_cf * std
```

#### HHI 집중도 지수

```python
def _calculate_hhi(self, weights: np.ndarray) -> float:
    """
    HHI (Herfindahl-Hirschman Index):
    포트폴리오 집중도 측정 지표.
    
    HHI = Σ(비중²)
    - 단일 종목 100% → HHI = 1.0 (완전 집중)
    - 10종목 동일 비중 → HHI = 10 × (0.1²) = 0.10
    - 낮을수록 분산 투자
    """
    return float(np.sum(weights ** 2))
```

## 9. 포트폴리오 최적화

### 📄 `app/portfolio/optimizer.py` — 포트폴리오 최적화

**목적:** 여러 종목에 얼마씩 투자해야 최적인지 수학적으로 계산합니다.

#### 평균-분산 최적화 (Markowitz)

```python
class MeanVarianceOptimizer:
    """
    Harry Markowitz의 현대 포트폴리오 이론 (1952년, 노벨상).
    "기대 수익률을 최대화하면서 위험(분산)은 최소화하는" 최적 비중 찾기.
    
    수학적으로는 scipy의 minimize() 함수로 풀기.
    """
    
    def optimize(
        self,
        returns: pd.DataFrame,    # 각 종목별 일별 수익률
        objective: str = "max_sharpe",  # 목표: 최대 샤프, 최소 분산, 위험균등
    ) -> np.ndarray:
        
        n = len(returns.columns)  # 종목 수
        
        # 입력 계산
        mean_returns = returns.mean().values * 252      # 연간 기대수익률
        cov_matrix = returns.cov().values * 252         # 연간 공분산 행렬
        
        # 제약 조건
        constraints = [
            {"type": "eq", "fun": lambda w: np.sum(w) - 1},  # 비중 합 = 1 (100%)
        ]
        bounds = tuple((0, 1) for _ in range(n))  # 각 비중 0~1 (공매도 없음)
        
        # 초기값: 동일 비중
        x0 = np.array([1/n] * n)
        
        if objective == "max_sharpe":
            # 샤프비율 최대화 = 음의 샤프비율 최소화
            def neg_sharpe(weights):
                port_return = weights @ mean_returns
                port_vol = np.sqrt(weights @ cov_matrix @ weights)
                return -(port_return - 0.02) / port_vol  # 음수 반환 (최소화용)
            
            result = minimize(neg_sharpe, x0, bounds=bounds, constraints=constraints)
        
        elif objective == "risk_parity":
            # 위험 균등 배분: 각 종목이 동일한 위험 기여도를 갖도록
            def risk_parity_obj(weights):
                cov_w = cov_matrix @ weights
                # 각 종목의 위험 기여도
                risk_contribs = weights * cov_w / (weights @ cov_w)
                # 모두 1/n 이 되어야 함 → 차이의 제곱합 최소화
                target = 1 / n
                return np.sum((risk_contribs - target) ** 2)
            
            result = minimize(risk_parity_obj, x0, bounds=bounds, constraints=constraints)
        
        return result.x  # 최적 비중 배열
```

#### 블랙-리터만 모델

```python
class BlackLittermanOptimizer:
    """
    Black-Litterman (1991): Markowitz의 개선 모델.
    
    문제: Markowitz는 순수 수익률 데이터만 사용 → 극단적인 비중이 나옴
    해결: 시장 균형(prior) + 투자자 견해(views)를 결합
    
    수식: 사후 수익률 = [(τΣ)^-1 + P'Ω^-1P]^-1 × [(τΣ)^-1π + P'Ω^-1q]
    
    여기서:
    - π (파이): 시장 균형 수익률
    - P: 견해 행렬 (어떤 종목에 대한 견해인지)
    - q: 견해 수익률
    - Ω (오메가): 견해의 불확실성
    """
    
    def add_view(
        self,
        symbol: str,
        expected_return: float,
        confidence: float = 0.5,
    ) -> None:
        """
        투자자 견해 추가.
        
        예: "AAPL이 연간 15% 수익을 낼 것 같다 (신뢰도 70%)"
        """
        self._views.append({
            "symbol": symbol,
            "expected_return": expected_return,
            "confidence": confidence,
        })
```

#### Redis Pub/Sub (발행-구독 패턴)

```python
"""
Pub/Sub 패턴:
  발행자(Publisher): 데이터를 채널에 보냄
  구독자(Subscriber): 관심있는 채널을 구독하여 데이터를 받음

마치 유튜브 채널 구독과 같음:
  업로더(=발행자)가 영상 올림 → 구독자들이 자동으로 알림 받음
"""

class RedisDataStream:
    
    async def publish_tick(self, symbol: str, tick: dict) -> None:
        """실시간 가격 데이터 발행"""
        channel = f"ticks:{symbol}"  # 채널 이름: "ticks:AAPL" 등
        message = json.dumps(tick)
        await self._redis.publish(channel, message)
    
    async def subscribe_ticks(self, symbol: str):
        """특정 종목의 실시간 틱 데이터 구독"""
        channel = f"ticks:{symbol}"
        pubsub = self._redis.pubsub()
        await pubsub.subscribe(channel)
        
        async for message in pubsub.listen():
            if message["type"] == "message":
                yield json.loads(message["data"])  # 데이터 하나씩 전달
```

#### Rate Limiting (속도 제한)

```python
async def is_rate_limited(self, key: str, max_count: int, window_seconds: int) -> bool:
    """
    슬라이딩 윈도우 방식의 속도 제한.
    최근 window_seconds 동안 max_count 이상 요청하면 차단.
    
    예: "1분에 100번 이상 API 호출 차단"
    """
    now = time.time()
    window_start = now - window_seconds
    
    pipe = self._redis.pipeline()
    
    # 1. 현재 시간을 집합에 추가 (score = timestamp)
    pipe.zadd(key, {str(now): now})
    
    # 2. 윈도우 밖의 오래된 항목 제거
    pipe.zremrangebyscore(key, "-inf", window_start)
    
    # 3. 현재 윈도우 내 카운트 조회
    pipe.zcard(key)
    
    # 4. 만료 시간 설정 (메모리 낭비 방지)
    pipe.expire(key, window_seconds)
    
    _, _, count, _ = await pipe.execute()
    
    return count > max_count  # True면 차단
```

#### SQLAlchemy ORM

```python
"""
ORM (Object-Relational Mapper):
파이썬 클래스 ↔ 데이터베이스 테이블을 자동으로 변환.
SQL을 직접 쓰지 않아도 됨.
"""
from sqlalchemy.orm import DeclarativeBase, mapped_column, Mapped
from sqlalchemy import String, Float, DateTime, JSON
import uuid

class Base(DeclarativeBase, AsyncAttrs):
    pass

class TimestampMixin:
    """created_at, updated_at 자동 관리"""
    created_at: Mapped[datetime] = mapped_column(default=func.now())
    updated_at: Mapped[datetime] = mapped_column(
        default=func.now(),
        onupdate=func.now()  # 수정 시 자동 갱신
    )

class OHLCVModel(Base, TimestampMixin):
    """
    주가 데이터 테이블.
    TimescaleDB 하이퍼테이블로 관리.
    """
    __tablename__ = "ohlcv"
    
    # UUID 기본키 (자동 생성)
    id: Mapped[uuid.UUID] = mapped_column(
        UUID(as_uuid=True),
        primary_key=True,
        default=uuid.uuid4  # 자동으로 고유 ID 생성
    )
    
    time: Mapped[datetime] = mapped_column(DateTime(timezone=True), nullable=False)
    symbol: Mapped[str] = mapped_column(String(20), nullable=False)
    timeframe: Mapped[str] = mapped_column(String(5), nullable=False)
    open:  Mapped[float] = mapped_column(Float, nullable=False)
    high:  Mapped[float] = mapped_column(Float, nullable=False)
    low:   Mapped[float] = mapped_column(Float, nullable=False)
    close: Mapped[float] = mapped_column(Float, nullable=False)
    volume: Mapped[float] = mapped_column(Float, nullable=False)
    
    # 복합 인덱스: (symbol, timeframe, time) 조합으로 빠른 검색
    __table_args__ = (
        Index("ix_ohlcv_symbol_tf_time", "symbol", "timeframe", "time"),
        {"timescaledb_hypertable": True}  # TimescaleDB 설정
    )
```

### 📄 `app/data/sql/timescale_schema.sql` — TimescaleDB 설계

```sql
-- TimescaleDB: 시계열 데이터에 특화된 PostgreSQL 확장
-- 수십억 행의 데이터도 빠르게 처리 가능

-- 1. 기본 테이블 생성
CREATE TABLE ohlcv (
    time        TIMESTAMPTZ NOT NULL,
    symbol      TEXT        NOT NULL,
    timeframe   TEXT        NOT NULL,
    open        DOUBLE PRECISION,
    high        DOUBLE PRECISION,
    low         DOUBLE PRECISION,
    close       DOUBLE PRECISION,
    volume      DOUBLE PRECISION
);

-- 2. 하이퍼테이블 변환 (TimescaleDB 핵심!)
-- 시간 기준으로 자동 파티셔닝 (7일 단위 청크)
SELECT create_hypertable('ohlcv', 'time', chunk_time_interval => INTERVAL '7 days');

-- 3. 공간 파티셔닝 (심볼별로 4개 파티션)
SELECT add_dimension('ohlcv', 'symbol', number_partitions => 4);

-- 4. 데이터 압축 설정 (저장 공간 70-90% 절약!)
ALTER TABLE ohlcv SET (
    timescaledb.compress,
    timescaledb.compress_segmentby = 'symbol, timeframe',
    timescaledb.compress_orderby = 'time DESC'
);

-- 30일 이전 데이터는 자동 압축
SELECT add_compression_policy('ohlcv', INTERVAL '30 days');

-- 5. 연속 집계 뷰 (미리 계산해서 빠른 조회)
-- 일봉을 실시간으로 집계
CREATE MATERIALIZED VIEW ohlcv_daily
WITH (timescaledb.continuous) AS
SELECT
    time_bucket('1 day', time) AS bucket,  -- 1일 단위로 묶기
    symbol,
    first(open, time) AS open,   -- 그날 첫 번째 시가
    max(high) AS high,           -- 최고가
    min(low) AS low,             -- 최저가
    last(close, time) AS close,  -- 마지막 종가
    sum(volume) AS volume        -- 총 거래량
FROM ohlcv
WHERE timeframe = '1m'  -- 1분봉에서 계산
GROUP BY bucket, symbol;
```

## 13. 이벤트 시스템

### 📄 `app/core/events.py` — 비동기 이벤트 버스

**목적:** 시스템의 각 부분이 서로 직접 연결되지 않고 이벤트를 통해 소통합니다.

#### 이벤트 버스 개념

```
전통적 방식:
  A → B.update()  (A가 B를 직접 호출)
  A → C.notify()  (A가 C를 직접 호출)
  → A가 B, C를 알아야 함 → 강한 결합

이벤트 버스 방식:
  A → EventBus.emit("신호생성")
                    ↓
  EventBus → B (구독자)
  EventBus → C (구독자)
  EventBus → D (구독자)
  → A는 아무것도 몰라도 됨 → 느슨한 결합
```

```python
class EventType(str, Enum):
    SIGNAL_GENERATED = "signal.generated"  # 신호 생성됨
    ORDER_PLACED     = "order.placed"      # 주문 접수됨
    ORDER_FILLED     = "order.filled"      # 주문 체결됨
    PRICE_UPDATED    = "price.updated"     # 가격 업데이트됨
    RISK_BREACH      = "risk.breach"       # 리스크 한도 초과!

class EventBus:
    
    def __init__(self):
        # 이벤트 타입별로 핸들러(처리 함수) 목록 저장
        self._handlers: dict[EventType, list[EventHandler]] = defaultdict(list)
    
    def on(self, event_type: EventType, priority: int = 5):
        """데코레이터: 이 함수를 특정 이벤트의 핸들러로 등록"""
        def decorator(handler):
            self.subscribe(event_type, handler)
            return handler
        return decorator
    
    async def emit(self, event: Event) -> None:
        """이벤트 발생시키기"""
        handlers = self._handlers[event.type]
        
        # 우선순위 순서로 실행
        for handler in sorted(handlers, key=lambda h: h.priority):
            await handler(event)

# 사용 예시
bus = EventBus()

@bus.on(EventType.SIGNAL_GENERATED)  # 신호 생성 이벤트 구독
async def handle_signal(event: Event):
    signal = event.data["signal"]
    print(f"새 신호: {signal.symbol} {signal.direction}")
    # 리스크 체크, 주문 생성 등 처리

# 이벤트 발생
await bus.emit(Event(
    type=EventType.SIGNAL_GENERATED,
    data={"signal": my_signal}
))
```

## 📚 주요 파이썬 패턴 정리

### SOLID 원칙 (이 프로젝트가 따르는 설계 원칙)

| 원칙 | 이름 | 내용 | 예시 |
|------|------|------|------|
| S | 단일 책임 | 클래스는 하나의 역할만 | RiskManager는 리스크만 관리 |
| O | 개방/폐쇄 | 확장은 쉽게, 수정 없이 | 새 전략 추가해도 기존 코드 수정 없음 |
| L | 리스코프 치환 | 하위 클래스는 상위 클래스 대체 가능 | YahooCollector, BinanceCollector는 BaseCollector 대체 가능 |
| I | 인터페이스 분리 | 작고 구체적인 인터페이스 | BaseDataCollector, BaseStrategy 각각 분리 |
| D | 의존성 역전 | 구체 클래스보다 추상에 의존 | DataManager는 BaseCollector 사용, 구체 클래스 직접 의존 안 함 |

### 자주 쓰는 패턴들

```python
# 1. 팩토리 패턴 (Strategy 생성)
def create_strategy(name: str, **params) -> BaseStrategy:
    strategies = {
        "ma_cross": MovingAverageCrossStrategy,
        "rsi": RSIStrategy,
        "macd": MACDStrategy,
    }
    cls = strategies.get(name)
    if not cls:
        raise ValueError(f"알 수 없는 전략: {name}")
    return cls(**params)

# 2. 싱글톤 패턴 (설정, DataManager)
@lru_cache(maxsize=1)
def get_settings() -> Settings:
    return Settings()

# 3. 컨텍스트 매니저 패턴 (성능 측정)
with PerformanceLogger("fetch"):
    data = await collector.fetch_ohlcv(...)

# 4. 데코레이터 패턴 (재시도)
@retry(stop=stop_after_attempt(3))
async def fetch_with_retry(): ...

# 5. 옵저버/이벤트 패턴 (EventBus)
@bus.on(EventType.SIGNAL_GENERATED)
async def handle_signal(event): ...
```

#### 단계 4: 리스크 체크 (신호가 있을 때)

```
Signal 수신
    │
    ▼
RiskManager.check_order(order, portfolio, portfolio_value)
    │
    ├─ ① 포지션 크기 체크 (10% 한도)
    │    ├─ 통과 → 다음 체크
    │    └─ 실패 → "포지션 크기 초과" 반환 + 주문 거부
    │
    ├─ ② 최대 낙폭 체크 (15% 한도)
    │    ├─ 통과 → 다음 체크
    │    └─ 실패 → "MDD 한도 초과" 반환 + 주문 거부
    │
    ├─ ③ 일일 손실 체크 (2% 한도)
    │    ├─ 통과 → 다음 체크
    │    └─ 실패 → "일일 손실 한도 초과" 반환 + 주문 거부
    │
    ├─ ④ 레버리지 체크 (1.5배 한도)
    │    └─ ...
    │
    └─ ⑤ 집중도 체크 (HHI)
         └─ ...
    │
    ▼
(통과, "") 또는 (실패, "이유") 반환
```

#### 단계 5: 백테스팅 실행 흐름

```
백테스팅 요청 수신
    │
    ▼
1. 과거 데이터 수집 (DataManager 사용)
    │
    ▼
2. 전략 신호 배열 생성 (strategy.generate_signal 반복)
   [0, 0, 1, 1, 1, -1, -1, 0, 1, ...] ← 각 날짜 신호
    │
    ▼
3. Numba JIT 계산 시작 (_calc_equity_curve)
   ┌─────────────────────────────────────┐
   │ for i in range(1, n):               │
   │   if signal[i] != position:         │
   │     청산 → 손익 계산 → 잔고 업데이트 │
   │     신규 진입 → 슬리피지 적용       │
   │   else:                             │
   │     미실현 손익 반영                │
   └─────────────────────────────────────┘
    │
    ▼
4. 성과 지표 계산
   ├─ 총 수익률 = (최종잔고 - 초기자본) / 초기자본
   ├─ 샤프 비율 = (연간수익률 - 무위험수익률) / 연간변동성
   ├─ 최대 낙폭 = (저점 - 고점) / 고점
   └─ 승률 = 이긴 거래 수 / 전체 거래 수
    │
    ▼
5. BacktestMetrics 반환
```

#### 단계 6: AI 예측 흐름

```
과거 데이터 입력
    │
    ▼
QuantFeatureEngineer.fit(train_data) ← 훈련 데이터로 기준값 설정
    ├─ 수익률 계산 (1일~252일)
    ├─ RSI 계산 (7, 14, 21일)
    ├─ ATR 계산 (14일)
    ├─ 볼린저밴드 계산
    └─ 모멘텀 계산 등 40여 개 특성
    │
    ▼
QuantFeatureEngineer.transform(test_data) ← 테스트 데이터 변환
    │
    ▼
StackingAlphaModel.fit(X_train, y_train) ← 모델 학습
    ├─ TimeSeriesSplit으로 5-fold 교차검증
    ├─ XGBoostAlphaModel 훈련
    ├─ LightGBMAlphaModel 훈련
    └─ 메타 모델 훈련 (OOF 예측값으로)
    │
    ▼
model.predict(X_test) ← 예측
    └─ 각 날짜에 대해 상승 확률 반환 (0~1)
         > 0.5 → 매수 신호
         < 0.5 → 매도 신호
```

#### 📈 전략 신호 받기

```python
import asyncio
from datetime import datetime

async def get_signal():
    from app.data.collectors.yahoo_collector import YahooFinanceCollector
    from app.core.types import Timeframe
    from app.strategies.technical_strategies import EnsembleStrategy, MovingAverageCrossStrategy, RSIStrategy, MACDStrategy
    
    # 1. 데이터 수집
    collector = YahooFinanceCollector()
    data = await collector.fetch_ohlcv("AAPL", Timeframe.ONE_DAY, datetime(2022, 1, 1))
    
    # 2. 앙상블 전략 생성
    strategy = EnsembleStrategy(
        strategies=[
            MovingAverageCrossStrategy(fast_period=20, long_period=50),
            RSIStrategy(period=14),
            MACDStrategy(),
        ],
        min_agreement=0.5,  # 50% 이상 동의 시 신호
    )
    
    # 3. 신호 생성
    signals = strategy.generate_signal(data)
    
    if signals:
        s = signals[0]
        direction = "🔼 매수" if s.is_long else "🔽 매도"
        print(f"신호: {direction}, 강도: {s.strength:.1%}, 신뢰도: {s.confidence:.1%}")
    else:
        print("📋 현재 신호 없음 (중립)")

asyncio.run(get_signal())
```

```python
import asyncio
from datetime import datetime

async def run_backtest():
    from app.data.collectors.yahoo_collector import YahooFinanceCollector
    from app.core.types import Timeframe
    from app.strategies.technical_strategies import MovingAverageCrossStrategy
    from app.backtesting.vectorized_engine import VectorizedBacktestEngine
    
    # 1. 5년치 데이터 수집
    collector = YahooFinanceCollector()
    data = await collector.fetch_ohlcv("AAPL", Timeframe.ONE_DAY, datetime(2019, 1, 1))
    
    # 2. 전략 설정
    strategy = MovingAverageCrossStrategy(fast_period=20, long_period=50)
    
    # 3. 백테스팅 실행
    engine = VectorizedBacktestEngine()
    metrics = await engine.run(
        strategy=strategy,
        data=data,
        initial_capital=100_000_000  # 1억원
    )
    
    # 4. 결과 출력
    print(metrics.to_report())
    # 출력 예시:
    # =========================================
    # 백테스팅 결과: MovingAverageCrossStrategy
    # =========================================
    # 총 수익률:      +34.21%
    # 연간 수익률:    +6.12%
    # 샤프 비율:      1.23
    # 최대 낙폭:      -18.45%
    # 승률:           58.3%
    # 총 거래 횟수:   47회
    # =========================================

asyncio.run(run_backtest())
```

#### 🤖 AI 모델로 예측하기

```python
import asyncio
from datetime import datetime

async def ai_predict():
    from app.data.collectors.yahoo_collector import YahooFinanceCollector
    from app.core.types import Timeframe, OHLCVFrame
    from app.ml.features.feature_engineering import QuantFeatureEngineer
    from app.ml.models.ensemble_model import StackingAlphaModel
    
    # 1. 충분한 데이터 수집 (AI 학습엔 최소 2~3년 필요)
    collector = YahooFinanceCollector()
    data = await collector.fetch_ohlcv("AAPL", Timeframe.ONE_DAY, datetime(2020, 1, 1))
    
    # 2. 훈련/테스트 분할 (80% 훈련, 20% 테스트)
    split = int(len(data.data) * 0.8)
    train = OHLCVFrame("AAPL", Timeframe.ONE_DAY, data.data.iloc[:split])
    test  = OHLCVFrame("AAPL", Timeframe.ONE_DAY, data.data.iloc[split:])
    
    # 3. 특성 엔지니어링
    engineer = QuantFeatureEngineer()
    engineer.fit(train)                        # 훈련 데이터로 기준값 계산
    
    X_train = engineer.transform(train).dropna()
    X_test  = engineer.transform(test).dropna()
    
    # 4. 레이블 생성 (5일 후 수익률 > 0이면 1, 아니면 0)
    import numpy as np
    future_returns = data.data["close"].pct_change(5).shift(-5)
    y = (future_returns > 0).astype(int)
    y_train = y.iloc[:split].loc[X_train.index]
    
    # 5. AI 모델 학습
    model = StackingAlphaModel()
    model.fit(X_train, y_train)
    
    # 6. 예측
    predictions = model.predict(X_test)
    print(f"최근 예측 (상승 확률): {predictions[-1]:.1%}")
    print("0.5 이상이면 상승 예측, 미만이면 하락 예측")

asyncio.run(ai_predict())
```

## 18. ❓ 문제 해결 가이드

### 오류 1: `ModuleNotFoundError: No module named 'xxx'`

**원인:** 라이브러리가 설치 안 됨  
**해결:**

```bash
# xxx 부분에 모듈 이름 입력
pip3 install xxx

# 예시:
pip3 install yfinance
pip3 install finance-datareader
pip3 install pykrx
pip3 install tenacity
```

---

### 오류 2: `numpy` 버전 충돌 경고

```
UserWarning: pykrx requires numpy < 2.0 but numpy 2.x is installed
```

**원인:** pykrx는 오래된 버전의 numpy를 요구함  
**해결:** 이 경고는 무시해도 됩니다. 대부분의 기능은 정상 작동합니다.  
완전히 해결하려면:

```bash
pip3 install "numpy<2.0"
```

---

### 오류 3: `ConnectionError: Binance 연결 실패`

**원인:** 인터넷 연결 문제 또는 방화벽  
**해결:**

```bash
# 인터넷 연결 확인
```
